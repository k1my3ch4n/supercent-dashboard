---
name: staging
description: 진행 작업을 나누어서 스테이징 하고, 커밋 메세지를 추천해 줍니다.
---

1. 현재 진행된 작업의 파일들을 확인해서 , 관련된 작업끼리 나누어 스테이징 합니다.
2. 스테이징된 작업에 대해서 커밋 메세지를 추천해 줍니다.
3. 커밋 메세지는 아래와 같은 형식으로 작성됩니다.
   - <type>: <subject>
   - <type>: feat, fix, docs, style, refactor, test, chore 중 하나

```
- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 스타일 변경 (포맷팅, 세미콜론 누락 등)
- refactor: 코드 리팩토링 (기능 변경 없음)
- test: 테스트 코드 추가/수정
- chore: 기타 변경사항 (빌드 스크립트 수정, 패키지 업데이트 등)
```

- <scope>: 변경된 파일이나 기능의 범위 (예: header, grid, api 등)
- <subject>: 변경사항에 대한 간단한 설명 (한글로 작성),

4. 하나의 스테이징이 끝나고 커밋 메세지를 추천한 후, `next` 명령어를 입력하면 다음 스테이징을 진행합니다.
