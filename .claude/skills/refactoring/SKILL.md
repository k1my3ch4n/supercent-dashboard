---
name: refactoring
description: 코드 리팩토링 작업을 수행하고, 변경된 내용을 보고합니다.
---

1. 이번 브랜치에서 변경된 파일들을 확인하고 리팩토링이 필요한 파일들을 식별합니다.
2. 식별된 파일들에 대해 리팩토링 작업을 수행합니다.
3. 리팩토링의 원칙은 아래와 같습니다.
   - 기능 변경 없이 코드의 구조와 가독성을 개선하는 작업입니다.
   - 함수 추출, 변수명 변경, 코드 중복 제거, 코드 구조 개선 등이 포함됩니다.
   - 파일은 최대한 작은 단위로 유지합니다.
   - 공통으로 분리할 수 있는 부분은 최대한 분리합니다.
   - constant, util, type 등으로 분리할 수 있는 부분은 최대한 분리합니다.
   - feature 의 크기는 최대한 작게 유지합니다.
4. 리팩토링 작업이 완료된 후, 변경된 내용을 요약하여 보고합니다.
   - 변경된 파일들의 목록
   - 주요 리팩토링 작업의 설명 (예: 함수 추출, 변수명 변경, 코드 구조 개선 등)
